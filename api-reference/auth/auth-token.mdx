---

## Overview

Use this endpoint to exchange your **consumer\_key** and **consumer\_secret** for a short-lived Bearer token. This is a server-to-server flow and does **not** require an existing Bearer token.

---

## Environments

### Sandbox

```text
https://dev.waftpay.io
```

### Production

```text
https://prod.waftpay.io
```

---

## Endpoint

```http
POST /auth/api/v1/generate-token
```

---

## Request body

```json
{
  "consumer_key": "APP_CONSUMER_KEY",
  "consumer_secret": "APP_CONSUMER_SECRET"
}
```

---

## Example response (success)

```json
{
  "status": true,
  "message": "Request processed successfully",
  "token": "abcdeFGHIjklmno@1234567890...",
  "expires_in": 7200000
}
```

Use the token in the `Authorization` header:

```http
Authorization: Bearer <token>
```

---

## Notes

- Use the credentials for the **specific product** you are calling (Payouts, Collections, Remittance, or Checkout).
- Sandbox credentials work only with the Sandbox environment (`https://dev.waftpay.io`).
- Production credentials work only with the Production environment (`https://prod.waftpay.io`).
- If you regenerate your keys, the old credentials stop working immediately.
